# 本番環境でのアカウント作成手順

本番環境（Railway）でユーザーアカウントを作成する方法を説明します。

## 前提条件

- Railwayにデプロイ済みであること
- データベース（MySQL）が正常に動作していること
- 環境変数（特に`DATABASE_URL`）が正しく設定されていること

## 方法1: Railway CLIを使用（推奨）

### 1. Railway CLIのインストール

```bash
npm i -g @railway/cli
```

### 2. Railwayにログイン

```bash
railway login
```

ブラウザが開き、Railwayアカウントでログインします。

### 3. プロジェクトをリンク

```bash
railway link
```

プロジェクトを選択します。

### 4. アカウント作成スクリプトを実行

#### 本部担当者アカウントを作成（全組織にアクセス可能）

```bash
railway run pnpm db:create-headquarters
```

このスクリプトは以下の情報を対話形式で入力します：
- メールアドレス
- 名前
- パスワード

本部担当者は全組織のデータにアクセスできます。

#### 通常の管理者アカウントを作成

```bash
railway run pnpm db:create-user
```

**注意**: このスクリプトは現在ハードコードされた値を使用しています。本番環境で使用する場合は、スクリプトを編集してメールアドレス、名前、パスワードを変更してください。

#### 組織とユーザーを同時に作成

```bash
railway run pnpm db:create-org
```

このスクリプトは以下の情報を対話形式で入力します：
- 組織名
- メールアドレス
- 名前
- パスワード
- ロール（admin/editor/viewer/headquarters）

## 方法2: Railwayダッシュボードから実行

### 1. Railwayダッシュボードにアクセス

[railway.app](https://railway.app)にアクセスしてプロジェクトを開きます。

### 2. 「Deployments」タブを開く

プロジェクトの「Deployments」タブを開きます。

### 3. 「Run Command」をクリック

「Run Command」ボタンをクリックします。

### 4. コマンドを実行

以下のいずれかのコマンドを入力して実行します：

- **本部担当者アカウント作成**:
  ```
  pnpm db:create-headquarters
  ```

- **通常の管理者アカウント作成**:
  ```
  pnpm db:create-user
  ```

- **組織とユーザーを同時に作成**:
  ```
  pnpm db:create-org
  ```

### 5. 対話形式で情報を入力

スクリプトが実行されると、対話形式で以下の情報を入力します：
- メールアドレス
- 名前
- パスワード
- （組織作成の場合）組織名とロール

## アカウントタイプの説明

### 本部担当者（headquarters）

- **特徴**: 全組織のデータにアクセス可能
- **用途**: 本部で全組織を管理する場合
- **作成方法**: `pnpm db:create-headquarters`

### 管理者（admin）

- **特徴**: 特定の組織の管理者権限
- **用途**: 各組織の管理者アカウント
- **作成方法**: `pnpm db:create-user` または `pnpm db:create-org`

### 編集者（editor）

- **特徴**: データの編集が可能
- **用途**: データ入力担当者
- **作成方法**: `pnpm db:create-org` でロールを選択

### 閲覧者（viewer）

- **特徴**: データの閲覧のみ可能
- **用途**: データ確認のみ必要なユーザー
- **作成方法**: `pnpm db:create-org` でロールを選択

## トラブルシューティング

### データベース接続エラー

- Railwayダッシュボードの「Variables」タブで`DATABASE_URL`が正しく設定されているか確認
- MySQLサービスが起動しているか確認

### スクリプトが実行されない

- `pnpm`がインストールされているか確認
- `package.json`のスクリプトが正しく定義されているか確認

### 既存ユーザーが存在する場合

スクリプトは既存ユーザーを検出すると、パスワード更新の確認を求めます。
- `y`を入力するとパスワードを更新します
- `n`を入力すると処理をキャンセルします

## セキュリティに関する注意事項

1. **パスワード**: 強力なパスワードを使用してください
2. **JWT_SECRET**: 本番環境では必ず強力なランダムな秘密鍵を使用してください
3. **環境変数**: 機密情報は環境変数で管理し、コードに直接書かないでください
4. **アクセス制御**: 必要最小限の権限を持つアカウントを作成してください
